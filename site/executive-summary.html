Executive Summary

A rate-limit friendly macro data pipeline that lands raw public data once, then produces repeatable curated datasets for static dashboards.

What exists

  • Ingestion: BLS LABSTAT flat files + DataUSA Tesseract cubes → S3 raw buckets with sync state and change detection.
  • Processing: raw → processed CSV tables (idempotent; rerunnable without re-downloading sources).
  • Enrichment: processed/raw → curated JSON payloads consumed by a static website.

See Pipeline for run evidence and Architecture for the system diagram.

Why it matters

  • Inflation pressure: productivity vs compensation and unit labor costs (margin compression signal).
  • Labor supply: participation vs population composition (non-citizen share).
  • Mismatch: unemployment vs commuting frictions (structural vs cyclical slack).

Each metric is designed to be explainable, reproducible, and extensible to richer cubes (CBP, BEA IO, FAF).

Rate-limit posture

  • Min sync interval per DataUSA dataset (skip provider calls when data updates annually).
  • Content hashing to avoid duplicate uploads when payloads do not change.
  • Polite pacing between datasets/series + exponential backoff with best-effort Retry-After.

Config knobs: DATAUSA_DATASETS, DATAUSA_MIN_SYNC_HOURS, DATAUSA_DELAY_SECONDS, DATAUSA_FORCE_REFRESH.

Run it (LocalStack)

For interviews, run the end-to-end LocalStack refresh and generate dashboards in one pass:

docker compose up -d
source .env.shared
source .env.localstack
python tools/localstack_full_refresh.py

Open site/index.html locally, or deploy the FomcSiteStack to S3 for a hosted static site.

What’s next

  • Broaden DataUSA coverage: CBP for industry structure, BEA IO for pass-through, FAF for logistics constraints.
  • Make dashboards “sliceable”: pre-aggregate, ship Parquet, and query in-browser (DuckDB-WASM) when needed.
  • Add governance: dataset manifests, schema pinning, and provenance metadata per artifact.
