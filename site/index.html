What This Project Demonstrates

This site is the output of a hands-on AWS workshop that builds a complete serverless data pipeline from scratch. The pipeline:

  1. Ingests time series data from the Bureau of Labor Statistics (BLS) LABSTAT flat files and the DataUSA population API
  2. Detects changes by comparing BLS file timestamps against previously synced metadata — only downloading what's new
  3. Stores raw data in S3 with sync state tracking (JSONL logs + state snapshots)
  4. Processes data through an SQS-triggered Lambda that joins BLS productivity series with population data
  5. Publishes results as JSON files that this static site renders as interactive charts

AWS Services Used

Amazon S3

Raw data storage, sync state management, processed output, and static website hosting for this site.

AWS Lambda

Two functions: a data fetcher (scheduled ingestion) and an analytics processor (event-driven analysis).

Amazon SQS

Decouples S3 put events from analytics processing. Provides reliable, at-least-once delivery with dead-letter queue support.

Amazon EventBridge

Scheduled rule triggers the data fetcher Lambda on a cron schedule to check for BLS updates.

AWS CDK

All infrastructure defined as code in Python — three stacks for data, pipeline, and site resources.

AWS IAM

Least-privilege roles for each Lambda function, scoped to specific S3 buckets and SQS queues.
